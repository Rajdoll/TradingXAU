## Decisions

| Decision | Choice | Rationale |
|---|---|---|
| Module type | Pure utility function | Not a LangGraph node — called by agents that own state updates |
| Return type | `SwingPoints` NamedTuple (`highs`, `lows`) | Single call, typed, destructurable; supports `.iloc[-1]` and `.values` |
| Series index | Original DataFrame `DatetimeIndex` | Pattern detectors need both price and time position |
| Empty result | `pd.Series(dtype=float)` | Callers check `len(swings.highs) == 0`; no exceptions for missing swings |
| Extrema comparison | `np.greater` (highs), `np.less` (lows) | Strict inequality — no plateaus treated as swings |
| Column access | `df["High"]`, `df["Low"]` | H4 DataFrame always has flat columns after data fetcher MultiIndex strip |

## Module: `tools/swing_detector.py`

### Public interface

```python
SwingPoints = namedtuple("SwingPoints", ["highs", "lows"])

def detect_swings(df: pd.DataFrame, window: int = SWING_WINDOW) -> SwingPoints
```

`highs` and `lows` are `pd.Series` indexed by the DataFrame's `DatetimeIndex`, values are OHLCV prices (`High` for highs, `Low` for lows).

Returns empty `pd.Series(dtype=float)` for either side when no extrema found.

### Caller pattern (for agents in Tasks 1.5–1.10)

```python
df = pd.DataFrame(state["ohlcv_h4"])
swings = detect_swings(df)

# For h4_last_hh / h4_last_hl (tf_arbiter, Task 1.10):
h4_last_hh = float(swings.highs.iloc[-1])
h4_last_hl = float(swings.lows.iloc[-1])

# For pattern geometry (Tasks 1.5–1.7):
high_prices = swings.highs.values   # np.ndarray of swing high prices
low_prices  = swings.lows.values    # np.ndarray of swing low prices
```

## Implementation

### `detect_swings(df, window) -> SwingPoints`

```python
def detect_swings(df: pd.DataFrame, window: int = SWING_WINDOW) -> SwingPoints:
    highs_arr = df["High"].to_numpy()
    lows_arr  = df["Low"].to_numpy()

    high_idx = argrelextrema(highs_arr, np.greater, order=window)[0]
    low_idx  = argrelextrema(lows_arr,  np.less,    order=window)[0]

    swing_highs = df["High"].iloc[high_idx] if len(high_idx) else pd.Series(dtype=float)
    swing_lows  = df["Low"].iloc[low_idx]   if len(low_idx)  else pd.Series(dtype=float)

    return SwingPoints(highs=swing_highs, lows=swing_lows)
```

**How it works:**
1. Extract `High` and `Low` columns as numpy arrays
2. `argrelextrema(arr, np.greater, order=window)` returns indices where each value is strictly greater than `window` neighbours on each side — these are swing highs
3. `argrelextrema(arr, np.less, order=window)` — same logic for lows
4. Slice the original Series at those indices to preserve the DatetimeIndex
5. Return as `SwingPoints` NamedTuple

## Testing (`tests/test_swing_detector.py`)

### Fixture strategy

Synthetic price series with known peaks and troughs (no network, no mocks):

```python
def _make_swing_df(prices: list[float], freq: str = "4h") -> pd.DataFrame:
    index = pd.date_range("2026-01-01", periods=len(prices), freq=freq)
    highs = prices
    lows  = [p - 10 for p in prices]   # lows offset below highs by 10
    return pd.DataFrame({"High": highs, "Low": lows}, index=index)
```

Example series for `window=1`:
```
prices: [3100, 3200, 3100, 3050, 3100, 3250, 3100, 3000, 3100]
swing highs at idx 1, 5  → prices 3200, 3250
swing lows  at idx 3, 7  → lows   3040, 2990  (prices - 10)
```

### Tests (7)

- `test_detects_swing_highs` — assert `swings.highs.values` matches expected prices at known indices
- `test_detects_swing_lows` — assert `swings.lows.values` matches expected prices at known indices
- `test_returns_series_indexed_by_datetime` — assert `isinstance(swings.highs.index, pd.DatetimeIndex)`
- `test_empty_highs_on_monotonic_increase` — strictly increasing prices → `len(swings.highs) == 0`
- `test_empty_lows_on_monotonic_decrease` — strictly decreasing → `len(swings.lows) == 0`
- `test_window_parameter_controls_sensitivity` — `window=1` finds more swings than `window=5` on same data
- `test_last_high_and_low_via_iloc` — `float(swings.highs.iloc[-1])` and `float(swings.lows.iloc[-1])` return correct values (the `h4_last_hh` / `h4_last_hl` use case)
