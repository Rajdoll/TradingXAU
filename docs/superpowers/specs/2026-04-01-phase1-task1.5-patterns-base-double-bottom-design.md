## Decisions

| Decision | Choice | Rationale |
|---|---|---|
| `base.py` content | `PATTERN_REGISTRY` list | `pattern_scanner` iterates a single list; decoupled from individual module names |
| Registry population | Each module appends itself at import time | Zero-config; adding a pattern = importing its module |
| Detector signature | `(df, swings, current_price) -> Optional[PatternResult]` | `swings` pre-computed once by scanner; `df` available for future geometry; `current_price` from state |
| Low tolerance | 1.5% (`abs(l1-l2)/min(l1,l2) <= 0.015`) | Tight enough to avoid false positives; loose enough for real market noise |
| Neckline | Highest swing high **between** the two lows (by timestamp) | Defines the resistance level the price must break |
| Entry zone | `neckline` to `neckline * 1.003` | Breakout entry: buy at/just above neckline; 0.3% chase limit |
| Invalidation | `min(low1, low2) - SL_BUFFER_PIPS * PIP_VALUE` | Pattern fails if either bottom breaks; buffer from `core.config` |
| `formation_pct` | `min(current_price / neckline, 1.0)` | 1.0 = price at/above neckline (breakout complete) |
| Direction | Always `"BUY"` | Double Bottom is a bullish reversal pattern |

## Module: `patterns/double_bottom.py`

### Pattern description

Double Bottom (W-shape): two swing lows at approximately the same price level with a swing high (neckline) between them. Bullish reversal — entry on neckline breakout.

- **Win rate:** 93% (Tier 1, Astronacci Dragon Gold 100 Pips)
- **Direction:** BUY only

### Constants

```python
WIN_RATE  = 0.93
TIER      = 1
TOLERANCE = 0.015   # max relative difference between the two lows
```

### Detection algorithm

```python
def detect_double_bottom(
    df: pd.DataFrame,
    swings: SwingPoints,
    current_price: float,
) -> Optional[PatternResult]:
```

**Steps:**

1. **Minimum lows check:** `len(swings.lows) < 2` → return `None`
2. **Extract last two lows:** `low1 = swings.lows.iloc[-2]`, `low2 = swings.lows.iloc[-1]`
3. **Tolerance check:** `abs(low1 - low2) / min(low1, low2) > TOLERANCE` → return `None`
4. **Find neckline:** highest swing high with timestamp strictly between `swings.lows.index[-2]` and `swings.lows.index[-1]`; if none exists → return `None`
5. **Compute and return PatternResult:**

```python
neckline          = float(highs_between.max())
invalidation      = min(low1, low2) - SL_BUFFER_PIPS * PIP_VALUE
entry_zone_low    = neckline
entry_zone_high   = neckline * 1.003
formation_pct     = min(current_price / neckline, 1.0)

return PatternResult(
    name               = "Double Bottom",
    win_rate           = WIN_RATE,
    direction          = "BUY",
    formation_pct      = formation_pct,
    entry_zone_low     = entry_zone_low,
    entry_zone_high    = entry_zone_high,
    invalidation_level = invalidation,
    tier               = TIER,
    skip               = False,
)
```

### Finding highs between the two lows

```python
t1 = swings.lows.index[-2]   # timestamp of low1
t2 = swings.lows.index[-1]   # timestamp of low2
highs_between = swings.highs[(swings.highs.index > t1) & (swings.highs.index < t2)]
if highs_between.empty:
    return None
```

### Registry registration

```python
from patterns.base import PATTERN_REGISTRY
PATTERN_REGISTRY.append(detect_double_bottom)
```

## Testing (`tests/test_patterns.py`)

### Fixture strategy

Synthetic H4 DataFrame with a known W-shape (window=1):

```python
# prices:  [3200, 3100, 3200, 3100, 3200]
#           High  Low1  High  Low2  current
# window=1: swing lows at idx 1 (3100) and idx 3 (3100)
#           swing high at idx 2 (3200) — between the two lows
# neckline = 3200, low1 = low2 = 3100
# entry_zone_low = 3200, entry_zone_high = 3200 * 1.003 = 3209.6
# invalidation = 3100 - 20 * 0.01 = 3099.8

def _make_double_bottom_df() -> pd.DataFrame:
    prices = [3200.0, 3100.0, 3200.0, 3100.0, 3200.0]
    index = pd.date_range("2026-01-01", periods=5, freq="4h")
    return pd.DataFrame({"High": prices, "Low": prices}, index=index)
```

**Note:** Using `High=Low=prices` means both swing highs and lows are detected from the same column values at their respective extrema positions.
